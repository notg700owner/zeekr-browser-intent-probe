Chain Viability Test

Renderer To Shell / ADB Evidence

The page can compare visible Chromium versions and map exposed browser surfaces. UID, SELinux domain, privileged vendor permissions, process isolation, and IPC exposure require manual evidence from chrome:// pages, shell/ADB, or an authorised recon APK.

  • Auto: exact visible Chromium version and comparison against 124.0.6367.60, 124.0.6367.201, and 124.0.6367.207.
  • Manual: chrome://version, chrome://sandbox, chrome://gpu, command line flags, process list, UID, SELinux context, package permissions, and exposed IPC surfaces.
  • Goal: decide whether a renderer compromise would plausibly reach privileged browser, GPU/Viz, vendor IPC, shell, or ADB/debug surfaces.
  • New: a post-renderer boundary triage row separates positive risk indicators from mitigations, blockers, and required non-web evidence.

Patch Candidate Coverage

What This Probe Looks For

These checks do not attempt exploitation. They only identify exposed browser subsystems where exact Chromium build and vendor patch status matter.

  • ANGLE / WebGL graphics exposure
  • Dawn / WebGPU exposure
  • V8 / WebAssembly exposure
  • Visuals/compositor, canvas, image, codec, and WebAudio smoke paths
  • WebAudio, WebRTC, permissions, and device API surfaces
  • Sensors, hardware, File System Access, WebAuthn, WebCodecs, and network policy surfaces
  • Service worker, Cache API, iframe sandbox, and policy behavior
  • Cloudflare-observed request headers, network/TLS/client-hint metadata, and legacy Chrome timing APIs
